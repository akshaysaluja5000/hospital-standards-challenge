==================================================
11. ASSIGNMENT LOGIC
==================================================

Use this assignment logic only for FINAL TEST results:

- 60–69% → assign 1 remediation plan
- 50–59% → assign 2 remediation steps
- below 50% → assign 2 remediation steps plus reassessment before completion

Again:
this logic applies ONLY to final/post-test scores.
Never to diagnostic or drill results.

12. UI / LAYOUT FIXES
==================================================

The current UI is too small, cramped, and hard to read.

Fix readability substantially.

Requirements:
- increase max content width
- make the layout visibly larger on desktop
- enlarge all headings
- enlarge card titles and body text
- enlarge summary stat cards
- enlarge status chips and filter controls
- enlarge primary/secondary buttons
- increase vertical spacing between sections and cards
- reduce the feeling of tiny components floating in a large empty blue canvas
- optimize for desktop and tablet first, then responsive mobile

Design goals:
- easy to read at normal desktop zoom
- no miniature text
- no tiny tracker cards
- no tiny filters squeezed into one row
- empty state should look intentional, centered, and readable
