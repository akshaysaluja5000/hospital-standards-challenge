Reference library
You are also right that all plans should exist somewhere as a reference, but not dumped openly into the page. The current screen already feels text-heavy and structurally flat, so exposing the full library inline would make it worse.

The right move is:

Keep the tracker page focused on assigned plans only.

Add a separate Plan Library reference that is available on demand.

Surface it as a dropdown, drawer, or modal rather than a permanent open block.

Best pattern:

Add a top-right secondary control: Browse Plan Library.

Clicking it opens a right-side drawer or modal.

Inside, users can filter by:

Facility type: Hospital / ASC

Category or chapter

Score band: 60–69, 50–59, below 50

Each entry shows the preset plan name plus 1–2 lines of what it includes.

That is better than a dropdown if the content is large. A tiny dropdown works for 5 items. You do not have 5 items. You have a structured library by hospital categories and ASC chapters, so a drawer is cleaner than pretending a giant list is a dropdown.
